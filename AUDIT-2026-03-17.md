# Security & Code Audit — Fixes & Action Items

> Comprehensive security audit, code review, UX review, and business assessment completed 2026-03-17.

---

## FIXED IN CODE (Already Implemented)

These changes are in the codebase and ready to deploy after running the DB migration.

### 1. IDOR Vulnerability — Task Authorization (CRITICAL)
**Problem:** Any authenticated user could complete/skip/snooze ANY task in the system by guessing UUIDs. The complete, skip, and snooze endpoints never verified the task belonged to the user's home.

**Fix:** Created `src/lib/api/authorize.ts` with shared `authorizeTaskAccess()` helper that verifies the task belongs to one of the user's homes via the home_members table. Applied to all three endpoints.

**Files changed:**
- `src/lib/api/authorize.ts` (new)
- `src/app/api/tasks/[id]/complete/route.ts`
- `src/app/api/tasks/[id]/skip/route.ts`
- `src/app/api/tasks/[id]/snooze/route.ts`

---

### 2. Input Validation with Zod (CRITICAL)
**Problem:** Zero input validation on any API endpoint. Malicious payloads could insert invalid data, trigger DB errors, or exploit type coercion.

**Fix:** Added Zod schemas for every API input in `src/lib/api/schemas.ts`:
- `createTaskSchema` — validates name, category, priority, frequency
- `completeTaskSchema` — validates cost, time, notes
- `snoozeTaskSchema` — validates days (1-365)
- `inviteSchema` — validates email format
- `pushSubscribeSchema` — validates endpoint URL and keys
- `onboardingSchema` — validates entire onboarding payload (home, systems, appliances, task setups)

**Files changed:**
- `src/lib/api/schemas.ts` (new)
- All API route files updated to use validation

---

### 3. Rate Limiting (HIGH)
**Problem:** Zero rate limiting. Invite endpoint could be abused for email enumeration. Task creation could be spammed.

**Fix:** Created `src/lib/api/rate-limit.ts` — in-memory sliding window rate limiter with three presets:
- **Standard (GET):** 60 requests/minute per user
- **Write (POST):** 20 requests/minute per user
- **Sensitive (invites, onboarding):** 10 requests/minute per user

Applied to all endpoints via the `apiHandler` wrapper.

**Files changed:**
- `src/lib/api/rate-limit.ts` (new)
- `src/lib/api/handler.ts` (new) — wraps all handlers with auth + rate limit + error handling

---

### 4. Error Handling Wrapper (HIGH)
**Problem:** API routes had no try/catch. DB errors returned raw 500s with stack traces.

**Fix:** Created `apiHandler()` wrapper in `src/lib/api/handler.ts` that catches:
- Zod validation errors → 400 with structured details
- Unexpected errors → 500 with generic message + server-side console.error

All API routes now use this wrapper (except onboarding which has its own due to different auth pattern).

---

### 5. Email Enumeration Fixed (MEDIUM)
**Problem:** `POST /api/home/invite` returned `{ autoAccepted: true }` when user existed vs `{ autoAccepted: false }` — revealing whether an email is registered.

**Fix:** Both paths now return identical `{ success: true }` response.

---

### 6. Unique Constraint on home_members (MEDIUM)
**Problem:** No unique constraint on `(home_id, user_id)` — race conditions could create duplicate memberships.

**Fix:** Added `uniqueIndex("home_members_unique_idx").on(table.homeId, table.userId)` to schema. Migration generated at `drizzle/0000_round_anthem.sql`.

---

### 7. Legacy Backfill Code Removed (MEDIUM)
**Problem:** `get-user-home.ts` had runtime migration code that ran on every request for users without memberships, doing N individual inserts.

**Fix:** Removed the backfill entirely. If a user has no memberships, they get an empty array. The onboarding flow already creates the membership correctly.

---

### 8. One-Time Task Handling (MEDIUM)
**Problem:** Completing a one-time task still calculated a next due date and kept it active.

**Fix:** `complete/route.ts` now detects `one_time` frequency and sets `isActive: false` after completion.

---

### 9. Pagination on Task List (MEDIUM)
**Problem:** `GET /api/tasks` returned ALL tasks with no pagination.

**Fix:** Added `limit` (default 50, max 200) and `offset` query params. Response now includes `total`, `limit`, `offset` for client-side pagination.

---

### 10. Session Refresh on API Routes (MEDIUM)
**Problem:** Middleware skipped API routes entirely — Supabase session cookies were never refreshed on API calls.

**Fix:** Updated `src/lib/supabase/middleware.ts` to create a Supabase client and call `getUser()` on API routes too, refreshing the session cookie.

---

### 11. Dashboard Returns All Tasks (LOW)
**Problem:** Dashboard sliced overdue/upcoming to top 10 with no indication more existed.

**Fix:** Removed the `.slice(0, 10)` — dashboard now returns all overdue and upcoming tasks.

## DECISIONS NEEDED (Requires Human Input)

### D1. Apple Sign-In
Apple requires Apple Sign-In if you offer social login and want App Store approval. Supabase supports it natively.
- **Recommendation:** Add Apple Sign-In + email/password as alternatives

### D2. Monitoring & Error Tracking
No production error visibility currently.
- **Recommendation:** Sentry (free tier, Next.js SDK)

### D3. Test Coverage
Zero tests. Scheduling logic is most critical.
- **Recommendation:** Start with unit tests for scheduling + auth helpers (~2 hours)

### D4. CI/CD Pipeline
No GitHub Actions or automated checks.
- **Recommendation:** Lint + typecheck + tests on PR, auto-deploy on merge

### D5. Push Notifications (VAPID Keys)
Service worker registered but no server-side push sender exists. This is the core value prop.
- **Recommendation:** Prioritize this (~3-4 hours)

### D6. Offline Support
No offline capability for basements/garages with poor signal.
- **Recommendation:** Post-launch (~6-8 hours)

### D7. Redis-Backed Rate Limiting
Current rate limiter is in-memory (resets on deploy, doesn't work across serverless instances).
- **Recommendation:** Keep in-memory for now, Upstash Redis when traffic grows

### D8. CORS Configuration
No explicit CORS headers.
- **Recommendation:** Vercel defaults sufficient unless building third-party API

### D9. File Storage for Documents
Documents table exists but no upload mechanism.
- **Recommendation:** Supabase Storage (zero new infrastructure)

### D10. Monetization Strategy
No payment infrastructure.
- **Suggested tiers:**
  - Free: 1 home, 2 members, basic tasks
  - Pro ($5-8/mo): Unlimited homes, document storage, full history, contractor CRM
  - Property Manager ($15-25/mo): Unlimited properties, team management, reporting

# Task Sub-Groups & Category Restructure — V1

**Date:** 2026-04-30
**Status:** Approved

## Problem

Users see 50+ tasks in flat lists under 7 categories. Categories like Exterior & Structure have 26 tasks which feels overwhelming. Tasks need a second level of organization (sub-groups) to feel digestible.

## Goals

1. Add sub-group labels within categories to visually break up long task lists
2. Restructure categories from 7 to 8 (add Air Quality & Health)
3. Cut/merge/adjust tasks identified during review
4. Sub-groups are visual dividers, not collapsible — they expand with their parent category
5. Dashboard stays flat and urgency-based (no sub-groups there)

## Data Model

Add a `subgroup` field to each task template:

```typescript
interface TaskTemplate {
  // ... existing fields ...
  subgroup: string; // e.g., "fire_safety", "roof_gutters", "pool_hot_tub"
}
```

The `subgroup` is a display-only label used to visually group tasks within a category on the Tasks page. It is NOT a new DB column — it lives on the template and is copied to task instances at creation time.

Add `subgroup` column to `task_instances` table (varchar, nullable) so the UI can group without needing to look up the original template.

### Sub-group display in UI:
- Sub-group headers are visual dividers within an expanded category section
- Sub-group header shows: label + urgency badge (e.g., "2 overdue")
- Sub-groups are NOT independently collapsible — they expand with the category
- Sub-groups with overdue tasks sort to top within their category
- Categories with no sub-groups (Appliances, Air Quality) show tasks in a flat list
